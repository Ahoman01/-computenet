# ComputeNet Whitepaper v1

## Abstract

ComputeNet is an ownerless decentralized compute network where COMPUTE functions as native fuel for verified machine work. The network is designed to let anyone request, perform, verify, record and reward useful computation without relying entirely on centralized compute providers.

## Founding Thesis

Bitcoin made money mineable. ComputeNet makes useful machine work mineable.

The core belief is simple: in the AI age, compute becomes a primary economic resource. If compute is centralized, intelligence becomes centralized. ComputeNet exists to create a fair, open and verifiable compute layer where machine work can be coordinated by a neutral network.

## Native Asset

COMPUTE is the native fuel of ComputeNet.

COMPUTE is not introduced as a speculative promise. It is designed as the internal fuel used to reward verified compute work, route jobs, settle machine-work receipts and eventually coordinate decentralized compute markets.

## Supply

ComputeNet begins with a 21,000,000 COMPUTE hard cap.

Launch principles:

- No premine
- No ICO
- No insider allocation
- Open source from the outset
- Anyone can run a node
- Anyone can mine under the same rules

## Consensus

ComputeNet uses Proof of Useful Compute.

Instead of rewarding meaningless computation alone, ComputeNet rewards work linked to useful machine tasks. Early versions include summarisation, classification, simulations, bootstrap compute and local AI inference through Ollama. Future versions can support embeddings, model evaluation, agent execution, GPU workloads, scientific simulation and distributed inference.

## Compute Receipts

Every mined compute task generates a receipt. A receipt records:

- job ID
- job type
- miner address
- block height
- payload hash
- compute proof hash
- elapsed time
- verification status
- signature

Receipts are designed to become the evidence layer of the network.

## Network Architecture

Each node maintains:

- local chain
- wallet
- peer registry
- pending jobs
- receipts
- pending transfers

Nodes expose an API for:

- status
- raw chain state
- job submission
- mining
- peer registration
- receipt inspection
- sync

## Peer Sync

ComputeNet v1 introduces simple peer sync. Nodes can register peers and pull longer chain state, receipts, jobs and peer lists. This is not final consensus, but it is the first decentralization step.
